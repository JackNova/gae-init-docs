Making it more personal

In the model.py you should change the dafault brand_name of the Config model to Phone Book 

brand_name = ndb.StringProperty(default='Phone Book')

If you refresh your browser, you might not see the changes right away. That's because you'll have to clear the datastore to be able to read the default properties. In order to achieve that stop the web server and execute the run.py script it using the additional -f argument to reset the datastore. 

$ ./run.py -fs

Adding new Entity

We are building a phone book so all we need is the contact list. In the end of the model.py file add the following: 

class Contact(Base):
  user_key = ndb.KeyProperty(kind=User, required=True)
  name = ndb.StringProperty(required=True)
  email = ndb.StringProperty(default='')
  phone = ndb.StringProperty(default='')
  address = ndb.StringProperty(default='')

The user_key property will store the User's key to make the contact list personal for every user. The rest of the properties are self explanatory.
