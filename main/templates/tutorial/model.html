Making it more personal

In the model.py you should change the default brand_name of the Config model to Phonebook 

brand_name = ndb.StringProperty(default='Phonebook')

If you refresh your browser, you might not see the changes right away. You'll have to clear the datastore to be able to (re-)read the default properties. Stop the web server and execute the run.py script using the additional -f argument to clear/reset the datastore. 

$ ./run.py -fs

Adding new Entity

As we are building a personal phone book, we'll need to add the contact list. Add the following at the end of model.py: 

class Contact(Base):
  user_key = ndb.KeyProperty(kind=User, required=True)
  name = ndb.StringProperty(required=True)
  email = ndb.StringProperty(default='')
  phone = ndb.StringProperty(default='')
  address = ndb.StringProperty(default='')

The user_key property will store the User's key to make the contact list personal for every user. The rest of the properties are self explanatory.
