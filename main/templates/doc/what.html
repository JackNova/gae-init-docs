gae-init is nothing more than a collection of pretty awesome stack, organized in a small project to kickstart your next Google App Engine project.

If you are already created a few App Engine projects you are aware of the fact that you need to spend some time setting up the project, getting the latest libraries, copying from older projects snippets that you've written already many times and you have to do some more boring stuff. This project is solving exactly that problem. It comes with the most nessesery stuff that are needed in any modern web application. No more wasting time on dealing with tools and start working on your next big thing right away!

gae-init is always up to date with the latest libraries and following the best practices. Check out the full feature list, get started with the tutorial or simply fork it.

I'm Lipis and I'm the guy behind this project. Yes I've finished the Getting Started Tutorial and after that I've read most of the manual. This project wouldn't be here without the old fellas, Mr. Tim and Mr. Chris that contributed with ideas, code, RedBulls, other stuff and the pretty awesome logo that was made by Mr. Kostas. Also special thanks goes to gmist, joernhees and stefanlindmark for testing and contributing with their time, code and ideas.

Some may ask why not using something that is already there like gae-boilerplate or flask-appengine-template. The answer is that, among other many things, gae-boilerplate's base.html is huge, unreadable and a bit chaotic compare to gae-init's base.html and that flask-appengine-template doesn't even have the pretty useful user sign-in.
